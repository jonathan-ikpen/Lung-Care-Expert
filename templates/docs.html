LungCare Expert Documentation

This page provides an overview of the LungCare Expert project, its features, usage, and technical details.

Overview

LungCare Expert is an interactive, explainable, knowledge-based expert system for triaging lung infections. It provides both a structured interview and a chatbot interface to help users get an initial, rule-based assessment of possible lung conditions.

What Can LungCare Expert Diagnose?

This project is designed to help triage and suggest possible causes for a range of lung-related symptoms. Based on your answers, it can suggest the likelihood of the following conditions:

  • Community-acquired pneumonia
  • Acute bronchitis
  • COVID-19 / Viral lower respiratory infection
  • Tuberculosis (TB) suspicion
  • Asthma exacerbation (non-infectious)
  • Pulmonary embolism (PE) suspicion

Capabilities: The system can:

  • Identify and explain which symptoms and rules led to each suggestion.
  • Detect red-flag symptoms that may indicate a medical emergency (e.g., blue lips, confusion, very high fever, low oxygen).
  • Guide users through adaptive, step-by-step questions to reduce diagnostic uncertainty.
  • Provide clear, explainable advice and next steps based on the user's responses.
Note: This tool is for educational and triage support only. It does not replace professional medical evaluation.

Features

  • Explainable rules: See which symptoms triggered each suggestion and why.
  • Smart questions: Adaptive follow-ups reduce uncertainty quickly.
  • Red-flag detection: Immediate advice for emergency symptoms.
  • Modern UI: Responsive, accessible, and user-friendly design.

How to Use

  1. Start the app with python app.py.
  2. Open your browser and go to http://localhost:5000.
  3. Choose Start Interview for a step-by-step assessment, or Open Chatbot for a conversational experience.
  4. Follow the prompts and review the possible conditions and advice provided.

Project Structure

  • app.py – Main Flask application.
  • inference_engine.py – Core logic for rule-based inference.
  • knowledge_base.py – Symptom and condition definitions, rules, and synonyms.
  • static/ – CSS, JS, and image assets.
  • templates/ – HTML templates for all pages.

Customization

You can add or modify rules, symptoms, and conditions in knowledge_base.py to adapt the system to new requirements or medical knowledge.
